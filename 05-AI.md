## Ollama

Versão validada no ambiente: `0.12.6`.

Recursos detectados:

- CUDA
- Flash Attention
- modo de baixa VRAM

### Modelos instalados

- `qwen3:8b`
- `qwen3-coder:30b`
- `nomic-embed-text`

### Estratégia de uso

#### `qwen3:8b`

Modelo recomendado para uso cotidiano:

- conversas gerais;
- Linux e Docker;
- automações;
- busca web;
- RAG;
- tarefas rápidas de programação.

#### `qwen3-coder:30b`

Reservado para tarefas mais complexas de código e arquitetura. No hardware atual, somente parte das camadas cabe na GPU; a maior parte da inferência ocorre em CPU.

### Benchmark registrado

Teste direto no Ollama:

```bash
time docker exec -it ollama ollama run qwen3-coder:30b
```

Prompt:

```text
Explique em poucas palavras o que é Docker
```

Resultado total:

```text
real 0m42.126s
```

O teste demonstrou que o modelo responde diretamente pelo Ollama em cerca de 42 segundos. Uma demora próxima de 10 minutos observada no Open WebUI aponta para processamento adicional na interface, ferramentas ou integração, e não para falha básica do Ollama.

## Open WebUI

Versão validada: `0.10.2`.

- Migrações de banco concluídas sem erro.
- A arquitetura atual usa Workspace Models, Knowledge, Prompts, Skills e Tools.
- Não há uma área separada chamada Assistants.

### Problema em investigação

O Open WebUI apresentou chamadas de ferramenta inadequadas e latência muito superior ao acesso direto ao Ollama com o `qwen3-coder:30b`. O diagnóstico deve comparar simultaneamente os logs de `open-webui` e `ollama` durante a mesma requisição.
